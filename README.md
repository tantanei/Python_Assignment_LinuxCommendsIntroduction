## 1. `top` 

### 설명

`top` 명령어는 현재 실행 중인 프로세스와 시스템 자원 사용량을 실시간으로 확인하는 명령어이다.

CPU 사용률, 메모리 사용률, 실행 중인 프로세스 목록 등을 확인할 수 있다.

### 사용법

```bash
top
```

### 주요 항목

* PID: 프로세스 ID
* %CPU: CPU 사용률
* %MEM: 메모리 사용률
* COMMAND: 실행 중인 명령어 이름

### 참고

`top` 실행 중 `q`를 누르면 종료된다.

필자는 Git Bash에서 `top` 명령어를 직접 실행해 보았으나, `bash: top: command not found`라는 오류가 발생했다.

알고 보니 `top`은 리눅스 환경에서 기본적으로 사용되는 명령어이며, Windows Git Bash에서는 기본 제공되지 않을 수 있다.

## 2. `ps` 

### 설명

`ps` 명령어는 현재 실행 중인 프로세스 정보를 출력하는 명령어이다.

`top`은 실시간으로 계속 갱신되지만, `ps`는 명령어를 실행한 순간의 프로세스 상태를 보여준다.

### 사용법

```bash
ps
```

### 사용 예시

```bash
ps aux
ps -ef
```

### 주요 옵션

* `a`: 다른 사용자의 프로세스도 표시
* `u`: 사용자 중심 형식으로 출력
* `x`: 터미널과 연결되지 않은 프로세스도 표시
* `-e`: 모든 프로세스 표시
* `-f`: 전체 형식으로 자세히 표시

### 주요 항목

- `PID`: 프로세스 ID
- `PPID`: 부모 프로세스 ID
- `PGID`: 프로세스 그룹 ID
- `WINPID`: Windows에서 사용하는 프로세스 ID
- `TTY`: 프로세스가 연결된 터미널
- `UID`: 사용자 ID
- `STIME`: 프로세스 시작 시간
- `COMMAND`: 실행된 명령어

## 3. `jobs` 명령어

### 설명

`jobs` 명령어는 현재 셸에서 실행 중이거나 일시 정지된 작업 목록을 확인하는 명령어이다.

백그라운드에서 실행 중인 작업을 확인할 때 자주 사용한다.

### 사용법

```bash
jobs
```

### 사용 예시

```bash
sleep 100 &
jobs
```

### 관련 명령어

* `&`: 명령어를 백그라운드에서 실행
* `Ctrl + Z`: 실행 중인 작업을 일시 정지
* `fg`: 백그라운드 작업을 포그라운드로 가져옴
* `bg`: 일시 정지된 작업을 백그라운드에서 계속 실행

## 4. `kill` 명령어

### 설명

`kill` 명령어는 프로세스에 신호를 보내는 명령어이다.

일반적으로 특정 프로세스를 종료할 때 사용한다.

### 사용법

```bash
kill PID
```

### 사용 예시

```bash
kill 1234
kill -9 1234
kill -l
```

### 주요 시그널

* `SIGTERM` 또는 `15`: 정상 종료 요청
* `SIGKILL` 또는 `9`: 강제 종료
* `SIGINT` 또는 `2`: 키보드 인터럽트


## 5. 명령어 비교

| 명령어    | 역할          | 특징                        |
| ------ | ----------- | ------------------------- |
| `top`  | 프로세스 실시간 확인 | CPU와 메모리 사용량을 실시간으로 보여줌 |
| `ps`   | 프로세스 상태 출력  | 실행한 순간의 프로세스 상태를 보여줌      |
| `jobs` | 셸 작업 확인     | 백그라운드 작업이나 정지된 작업을 확인     |
| `kill` | 프로세스 제어     | PID를 이용해 프로세스에 신호를 보냄     |

---

## 6. 정리

`top`과 `ps`는 실행 중인 프로세스를 확인하는 명령어이다.

`top`은 실시간으로 상태를 보여주고, `ps`는 명령어를 실행한 순간의 상태를 출력한다.

`jobs`는 현재 셸에서 실행 중인 작업을 확인할 때 사용한다.

`kill`은 특정 프로세스를 종료하거나 제어할 때 사용한다.
